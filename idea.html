Overview

My project will be an illustration-heavy quiz-type program that asks users various questions to determine which game(s) they should play next based on a compatibility score. The compatibility score will be based on data that will be scraped from a large video game digital distribution service. This ensures that the data and games collected will be up-to-date and accurate. After these questions have been answered and the probability score has been calculated, the program will recommend the user games based on their results.

Tools and Technologies

In addition to a computer, the main technological components of this will be a web scraper, a database, and a UI interface. Some of these, such as the web scraper and the database, can be outsourced to third parties by using open-source programs. For instance, Scrapy could be used as the web scraper and MySQL for the database. However, there is also the potential to develop a web-scraper in-house with languages such as Java or Python. It is recommended that the UI interface be developed in house so to have more control over the design aesthetics, details, branding, and overall experience. This then ties into the illustrations, which will likely be completed in Adobe Illustrator, or a similar program.
